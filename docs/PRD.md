# PRD - Onchain Copilot（链上副驾驶）

## 1. 背景
链上操作门槛高，常见路径割裂：看行情、找机会、做交易、风控复盘分散在多个工具。
Onchain Copilot 的目标是把这条链路“对话化 + 可执行化”。

## 2. 目标用户
- Web3 活跃用户（有交易行为）
- 内容/运营从业者（需要快速把信息转成动作）
- DAO/团队财务（频繁链上支付）

## 3. 核心价值
- 降低操作复杂度：自然语言直达执行计划
- 提升执行质量：默认风控前置
- 强化可解释性：每步给出原因与参数

## 4. MVP 功能（V2 拆解）
1. 意图解析（交易 / 运营 / 支付）
2. 策略与计划生成：分批计划、热点内容、支付流程
3. 风控检查：仓位、滑点、Gas、黑名单、免责声明
4. 执行确认：高风险动作必须确认
5. 复盘与回执：计划 vs 实际、可推送

## 5. 赛题映射（OnchainOS）
- 钱包管理：读取资产、链余额、支付能力
- 行情解析：拉取行情/波动、热点趋势
- 交易执行：下单/Swap
- 链上支付：转账、Gas 控制、延后策略
- DApp 连接：协议交互、跨应用工作流

## 6. 成功指标（比赛向）
- 结合度：至少展示 3 个 OnchainOS 能力
- 实用性：真实可复用场景（非玩具）
- 可演示性：90 秒内完整走通“输入→执行→结果”

## 7. 风险与边界
- 不做无确认自动重仓交易
- 不承诺收益，不提供投资建议
- 关键动作必须显式确认
