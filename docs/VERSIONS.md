# 版本拆解（V2）

## 1. 交易版（Trading Agent）
- **目标**：让普通交易者一句话完成“场景 → 策略 → 风控 → 执行”。
- **亮点**：
  - 分批下单（TWAP/TCA）并可设置间隔、滑点、回撤停手。
  - 调用 OnchainOS 行情、钱包余额、交易执行能力。
  - 输出成交前置风控 + 执行确认 + 自动复盘。
- **关键指标**：
  - 计划生成时间 < 1s
  - 默认提供 3 条风控检查
  - 复盘摘要 30s 内可生成（支持推送）

## 2. 运营版（Ops Agent）
- **目标**：你擅长的运营/内容场景——把热点捕捉、内容生成、交易观察三件事一次完成。
- **流程**：
  1. 抓取链上与社媒热点（OnchainOS 行情 + 情绪 API）。
  2. 生成 X/Twitter 可发文案 + CTA。
  3. 自动衔接成“可监控的交易观察列表”。
- **亮点**：
  - 运营与交易协同：热点 ≈> 内容 ≈> 观察 ≈> 决策。
  - 默认附带事实核验提示 & 免责声明。

## 3. 支付版（Payment Agent）
- **目标**：让链上转账 / 支付更安全，gas、合规、优先级一目了然。
- **流程**：
  1. Quote：读取 OnchainOS 当前 gas、token 价格。
  2. Compliance：地址黑名单、防诈骗提示。
  3. 执行：设置优先级、Gas Cap、延后/立即策略。
- **亮点**：
  - 自动生成支付回执与记账条目。
  - gas 超阈值 → 支持延后或部分执行。

## 场景对比
| 维度 | 交易版 | 运营版 | 支付版 |
| --- | --- | --- | --- |
| 用户 | 交易者 | 运营/新媒体 | 财务/团队协作 |
| 输入形式 | 交易意图 | 热点 + 内容需求 | 转账指令 |
| 核心输出 | 分批执行计划 | 热点 + 内容 + 观察 | 支付流程 + 风控 |
| OnchainOS 能力 | 行情 / 钱包 / 交易 / DApp | 行情 / 内容 / Watchlist | 钱包 / 支付 / 风控 |
| 二次确认 | 必选 | 可选 | 必选 |

## 为什么这个题合适
- **覆盖 OnchainOS**：钱包、行情、交易、支付、DApp 能力均可展示。
- **评审友好**：90s 视频即可跑通自然语言到执行，视觉冲击力强。
- **可持续**：不是一次性 Demo，能沉淀为真实工具，持续演进。
